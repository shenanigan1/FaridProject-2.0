## Stack
- Django + DRF
- Angular
- Tailwind

## Mandatory delivery order
For every feature, bugfix, or improvement, always follow this order:
1. Plan
2. Impacted files
3. Tests first
4. Implementation
5. Edge cases
6. Corrections/refactor if needed

## Rules
- Think before coding
- Use clean, maintainable, scalable code
- Strong security and permissions
- Thin Django views, business logic in services
- Strong serializers, permissions, and API tests
- Small Angular components and services
- Typed code, no unnecessary `any`
- Keep template logic minimal
- Use guards, interceptors, reactive forms, and feature-based structure
- Always show the file map of created/modified files
- Reuse existing files when possible instead of inventing new ones
- Be explicit about assumptions
