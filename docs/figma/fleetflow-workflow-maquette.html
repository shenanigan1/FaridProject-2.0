Règles UX / sécurité à appliquer dans l’app

À transformer en composants shared

Workflow

  • Un écran dédié par décision importante.
  • Plus de boutons qui font la même chose.
  • Les filtres `/tests` deviennent la source unique de recherche.
  • Chaque état test est visible: draft, progress, review, validated, rejected.

Accessibilité AAA

  • Contrastes forts et états focus visibles.
  • Labels explicites sur tous les champs.
  • Messages d’erreur compréhensibles.
  • Décisions critiques confirmées et non uniquement colorées.

Architecture

  • App shell et nav par rôle centralisés.
  • Cards, tabs, chips, sheets, buttons en shared.
  • Pages alimentées API ou empty state propre.
  • Scoping backend conservé: manager, candidat, RH/admin.
